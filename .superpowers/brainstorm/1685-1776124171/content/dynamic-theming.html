Flujo de implementación

1
Base de datos — Prisma
Añadir accentColor String? al modelo Business. Al subir logo → backend extrae color dominante con colorthief y lo guarda.
↓
2
Tailwind config — apps/admin y apps/saas
En tailwind.config.ts: colors.primary: 'var(--brand-primary)'. Así bg-primary, text-primary, border-primary responden automáticamente.
↓
3
ThemeProvider — inyección de variable CSS
El layout lee accentColor del tenant via API y hace document.documentElement.style.setProperty('--brand-primary', color). Fallback: #7c3aed.
↓
4
Restricción de fondo fijo
--brand-primary solo se aplica a: botones (bg-primary), borders activos (border-primary), focus rings, badges activos, links y nav activo. El fondo #080810 siempre fijo en CSS global.
Restricción visual cumplida: Los 4 ejemplos arriba usan el mismo fondo oscuro #080810. Solo cambian botón de acción, campo enfocado, nav activo y badge de plan — elementos de estado e interacción. Nunca el fondo ni las superficies de cards.
